## 2. 핵심 발견 (다음 세션에서 중요)

**seCall 은 현재 사실상 libkiwi 0.22.x 전용이다. 0.23.x 는 init 에서 SEGFAULT.**

- #144 의 FFI 회귀 테스트를 만들다 실제 버그를 잡음: fresh 환경에 정품 libkiwi **0.23.2** 를
  provision 해서 seCall 의 kiwi 경로를 태우면 `KiwiTokenizer::new()` -> `Kiwi::new()`
  (builder/init 경로)에서 하드 크래시(STATUS_ACCESS_VIOLATION).
- 같은 lib+model 로 번들 `kiwi-cli-0.23.2` 는 정상 분석(`아버지/NNG 가/JKS 방/NNG ...`) ->
  lib/model 은 무결, **kiwi-rs 바인딩의 init 경로 0.23 비호환**.
- probe 로 crash 를 analyze 가 아니라 **init** 으로 확정. builder 경로 시그니처
  (`kiwi_builder_init` 4-arg 등)는 0.23.2 capi.h 와 일치하는데도 죽으므로, 모델 로딩/빌드옵션
  레벨의 0.23 대응 문제로 추정(상류 kiwi-rs 조사 영역).
- #143 은 analyze-path `KiwiAnalyzeOption` ABI 만 고쳤고, 이 init-path 비호환은 미해결.
- 이 환경 실측: 설치된 `%LOCALAPPDATA%\kiwi\lib\kiwi.dll` = **0.22.2**(12MB, 2025-12-15).
  이전 핸드오프의 "0.23.2 설치" 기록은 틀렸음. 0.22.2 는 48B struct 의 32B prefix 만 읽혀 안전 ->
  그래서 사용자/CI 는 무사했음.
- 그래서 #144 의 핀은 `v0.23.2` 가 아니라 **실증된 `v0.22.2`** 로 확정. `tokenizer.rs` 의
  `KIWI_LIBKIWI_TAG` 상수 주석에 "0.23+ 로 bump 금지" 경고 명시.

**별개 포크 버그**: `bootstrap.rs::find_asset_url` 가 릴리스 JSON 의 중첩 `uploader` 객체 때문에
brace 매칭이 틀어져 asset URL 을 못 찾음 -> libkiwi 미설치 fresh 설치자 전원에게 auto-download
실패. 포크 유닛테스트는 `uploader` 없는 평탄한 mock 만 써서 못 잡음. #144 CI 는 이 다운로더를
안 쓰고 인증된 `gh release download` + `KIWI_LIBRARY_PATH`/`KIWI_MODEL_PATH` 로 직접 provision.

메모리: `secall_libkiwi_version` 저장(0.23+ bump 금지 제약 + 재현 방법).

## 5. 인프라/환경 상태 (중요)

- **설치 바이너리** `D:\.cargo\bin\secall.exe` = **v0.7.0 릴리즈본**. #142(임베딩 가드)/#143/#144
  는 아직 미포함. 최신 main 반영하려면 재빌드+재설치 필요(급하지 않으면 다음 릴리스 대기).
- **libkiwi 재현/검증 방법**: `gh release download vX -R bab2min/Kiwi -p 'kiwi_win_x64_vX.zip'
  -p 'kiwi_model_vX_base.tgz'` -> 압축해제 -> `KIWI_LIBRARY_PATH`(…/lib/kiwi.dll|libkiwi.so) +
  `KIWI_MODEL_PATH`(…/models/cong/base) 지정 -> `SECALL_KIWI_FFI_TEST=1 cargo test -p secall-core
  --lib test_kiwi_ffi_abi_smoke`. (Git Bash tar 는 `--force-local` 필요, Windows 경로 `C:` 오인)
- **DB**: `%LOCALAPPDATA%\secall\index.sqlite` (.db 아님). 메모리 `secall_db_path`.
- **8080 serve**: 현재 미기동(재부팅으로 종료). 필요 시 `secall serve --port 8080`.
- **MCP 정리**: `pencil` 서버 제거 + `greptile` 플러그인 비활성화(둘 다 연결 실패 상태였음). 다음
  세션부터 적용. 남은 MCP: gitea, planka, secall, supabase, tunallama.

## 7. 규명/학습 (재작업 방지)

- **libkiwi 버전 함정**: 설치본 버전을 파일 크기/날짜로 검증할 것(0.22.2=12MB/2025-12-15,
  0.23.2=23MB). "설치돼 있고 동작한다"가 곧 "0.23 호환"을 뜻하지 않음(0.22.x 라 무사했을 뿐).
- **FFI 회귀 테스트의 가치**: 헤더 시그니처 대조만으로는 못 잡는 init-path 크래시를 실제 lib 로
  태워서 잡음. 헤더 일치 != 런타임 호환.
- **CI 에서 native lib 검증**: 런타임 auto-download 의존 금지(rate-limit + 포크 파서 버그). 인증된
  `gh release download` + env 로 직접 provision 이 결정적.
