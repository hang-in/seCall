### 2.2 웹 세션 로딩 검토 — `docs/reviews/web-session-loading-review.md`
- 커밋 `b859e3b`. "세션 로그를 웹에서 레이지로딩으로 다 로딩 가능? RAM?" 질문에 대한 코드 검토.
- **결론**: 데이터는 **이미 lazy**(리스트=`useInfiniteSessions` 페이지네이션 무한스크롤 + `useInfiniteScroll` IntersectionObserver, 본문=`useSession(id,full)` 온디맨드). preload-all 아님 → 데이터 측 RAM 압박 없음.
- **병목 2건**: ① 리스트 **DOM 가상화 부재**(`SessionList` 가 `allItems` 전량 `.map` → 스크롤 누적) ② 긴 본문 **통째 렌더**(`MarkdownView` transcript 전체 react-markdown).
- **권장**: P0 = 리스트 가상화(`@tanstack/react-virtual`, 이미 react-query 사용) · P1 = 긴 본문 `content-visibility:auto` + turn 청크 무한스크롤 · P2 = detail 쿼리 `gcTime` 바운딩.
