## 3. 다음 작업

### 3.1 (최우선, 사용자 지정) 배포방법 고도화

**현재 배포 방식 (출발점):**
- **원라인 설치**: `install.sh`(Unix) / `install.ps1`(Windows) — GitHub release 의 prebuilt 바이너리를 받아 `~/.local/bin` 에 설치. `install.sh` 는 `uname` 으로 타깃 판별.
- **릴리스 자동화**: `.github/workflows/release.yml` — `v*` 태그 push 시 → web/dist 빌드 → `cargo build --release` 로 **3 타깃만**(aarch64-apple-darwin, x86_64-apple-darwin, x86_64-pc-windows-msvc) → tar.gz/zip 패키징 → `softprops/action-gh-release` (generate_release_notes). Windows 는 onnxruntime.dll 수동 번들.
- **cargo install --git** (#104 에서 build.rs placeholder 로 빌드 실패 복구).
- release profile: opt-level 3, lto "fat", codegen-units 1, strip.

**고도화 후보 (다음 세션에서 사용자와 범위 확정 — 아직 미결정):**
1. **Linux 바이너리 부재** ⚠️ 가장 큰 공백 — release.yml 에 Linux 타깃 없음. `install.sh` 의 Linux 경로는 prebuilt 없이 **소스 컴파일 fallback**(`cargo install`). → `x86_64-unknown-linux-gnu` / `aarch64-unknown-linux-gnu` (또는 musl 정적) 추가.
2. **패키지 매니저 부재** — Homebrew tap / formula, `cargo-binstall` 메타데이터(Cargo.toml `[package.metadata.binstall]`), Windows scoop/winget. binstall 은 메타데이터만 추가하면 기존 release 자산 재사용 가능 → 저비용.
3. **체크섬/서명 부재 (supply-chain)** — release 자산에 SHA256SUMS 없음, `install.sh` 가 검증 없이 다운로드. → 체크섬 생성 + install 스크립트 검증.
4. **버전 핀 옵션** — `install.sh` 가 항상 latest. `SECALL_VERSION` env 지원 검토.
5. **macOS 미서명/미공증** — Gatekeeper quarantine 경고. 서명/notarize 는 Apple 개발자 계정 필요 → 비용 대비 검토 (범위 밖일 수 있음).
6. (선택) Docker 이미지 / aarch64 Linux(ARM 서버·OpenVINO 사용자).

> 권장 시작점: **#1 Linux 바이너리 + #2 cargo-binstall** 이 효과 대비 비용이 가장 낮음. #3 체크섬은 보안상 같이 묶을 만함. 일괄 말고 단계적 PR.

## 5. 작업 방식 (반드시 지킬 것 — 직전 세션들에서 검증)

- **fork PR 머지 전 로컬 풀 검증**: fork PR 은 CI 자동 실행 안 됨. `cargo fmt --check` + `clippy --workspace --all-targets -- -D warnings` + `cargo test --workspace` 직접. (memory `feedback_local_ci_gate`, `feedback_fork_pr_merge_gate`)
- **검증과 commit/push 분리 배치**: 같은 Bash 배치에 넣으면 검증 실패해도 push 됨. (memory `feedback_verify_before_push_separate_batch`)
- **release 흐름**: PR(Cargo.toml bump + CHANGELOG) → 머지 → `git tag vX` push → `release.yml` 자동 트리거(빌드+자산+Release). 수동 gh release create 불필요.
- **windows CI stall**: clippy step 20분+ 멈추면 runner stall → `gh run cancel` 후 `gh run rerun`.
- destructive(branch -D / 파일 삭제 / 되돌리기 어려운 것)는 confirm. push/PR/머지는 위임받음.
- **bash 출력 토큰 마스킹 주의**: 이 환경은 rg/grep stdout 에서 vendor/model 문자열(예: gemini→`n`)을 마스킹함. 문자열 리터럴 확인·편집은 **Read 도구로 원문** 확인할 것.
