## 5. 작업 방식 — 이번 세션 학습 (반드시 지킬 것)

- **fork PR 머지 전 로컬 풀 검증**: CI 가 fork PR 에 자동 실행 안 됨. `fmt --check` + `clippy -D warnings` + `test` 직접. (memory `feedback_fork_pr_merge_gate`)
- **검증과 commit/push 분리**: 같은 Bash 배치에 넣으면 검증 실패해도 push 됨. FMT=0/CLIPPY=0/TEST=0 **눈으로 확인 후** 별도 배치로 commit/push. (memory `feedback_verify_before_push_separate_batch` — 이번 세션 P89 에서 4회 위반 후 교정)
- **CI watch 는 `gh run watch <id> --exit-status`** 로 결과를 명시 확인. `gh pr checks --watch` 는 exit 0 이 CI 성패와 무관하게 나옴 (오판 경험 있음). 머지 전 job conclusion 직접 확인.
- **windows CI stall**: clippy step 에서 20분+ 멈추면 runner stall — `gh run cancel` 후 `gh run rerun` (코드 무관, 재실행으로 해결됨).
- **release 흐름**: PR(Cargo.toml bump + CHANGELOG) → 머지 → `git tag vX` push → **release.yml 가 자동 트리거**되어 바이너리 빌드+자산 업로드+Release 생성 (수동 gh release create 불필요, body 만 CHANGELOG 로 보강). tag 가 main HEAD 가리키는지 확인.
- destructive(worktree remove/branch -D/run delete) 는 confirm. push/PR/머지는 위임받음.
