# 02. 에디터 (WYSIWYG + Markdown 소스)

## 2.1 두 가지 편집 모드 (상단 중앙 토글 2버튼)

- **WYSIWYG(리치)** ↔ **Markdown 소스(`M↓`)**. 같은 문서를 리치 편집/원문 편집으로 오감.
- **Markdown 소스 모드**: 줄번호 + 문법 하이라이트 + **코드 폴딩**(섹션 접기 chevron)되는 원문 에디터. (설정에 word-wrap 토글 있음)

## 2.2 WYSIWYG 본문 (Notion식 블록)

- **breadcrumb**(상단 좌) — 경로 네비.
- 블록: H1/H2/H3 · 문단 · **불릿 리스트** · **인라인 표**(헤더행 + 행/셀 인라인 편집, 행 hover `+`로 행/열 추가, 표 하단 `+`로 블록 추가) · 코드블록 등.
- **위키링크 인라인 렌더**: `[[research/note]]` → 파랑 밑줄 링크, 클릭 시 이동.

## 2.3 슬래시 `/` 블록 메뉴

- 커서에서 `/` → **"Basic blocks"** 메뉴: H1 / H2 / H3 · Bullet List · Ordered List · **Task List** · Quote · Code Block … (아래로 더 있음).
- **각 항목 hover 시 우측에 라이브 프리뷰 카드**(예 "Heading 1 / Big section heading").
- **구현 노트**: 슬래시 커맨드 + 항목별 프리뷰.

## 2.4 블록 hover 핸들 (Notion식)

- 블록 왼쪽에 `+`(아래 블록 추가) + `⁞⁞`(드래그 재정렬) 핸들 노출.

## 2.5 위키링크 자동완성 `[[` — ★P0 핵심

- 입력 즉시 **문서 + 폴더 제안 드롭다운**. 각 항목 = **제목 + 경로**(예 `리서치 노트 / research/note`). 퍼지 필터.
- 문서뿐 아니라 폴더·`.ok`도 후보.
- **구현 노트**: 링크 그래프의 입력단. seCall이 위키 상호참조를 살리려면 필수.

## 2.6 하단 상태바

- 좌: `프로젝트명  ⑂ 브랜치`(git). 우: `N words · N chars · ~N tokens` — **토큰 카운트까지** 표시(LLM 친화적, 세션·LLM 도구인 seCall에 잘 맞음).

## 2.7 seCall 적용

- **P0**: WYSIWYG 블록 에디터(TipTap/ProseMirror/Lexical 계열) + `[[wikilink]]` 실시간 렌더·자동완성.
- **P2**: 슬래시 커맨드, 블록 드래그 핸들, 토큰카운트 상태바(저비용).
- Markdown 소스 모드는 "원문 편집" 니즈 있으면 추가(폴딩·하이라이트).
