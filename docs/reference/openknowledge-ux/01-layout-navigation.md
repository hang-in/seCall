# 01. 레이아웃 · 네비게이션

## 1.1 전체 레이아웃 — 3-pane 워크스페이스

```
┌───────────┬───────────────────────────────┬──────────────┐
│ 좌: FILES │  중앙: 탭 + 에디터              │ 우: 컨텍스트 │
│ (트리·검색)│  (breadcrumb, 모드토글, 본문)  │ 패널(탭 4개) │
│           │  (하단 상태바)                 │              │
└───────────┴───────────────────────────────┴──────────────┘
```

- **좌/우 패널 접기 토글**: 중앙 상단 좌측 아이콘=좌패널, 우측 아이콘=우패널.
- 우상단 전역: **Share · Settings(⚙) · Help(?)**.
- **왜**: 문서 편집 중에도 좌(탐색)·우(맥락)를 동시에. seCall이 세션→위키 탐색을 한 화면에서 하려면 기본기.

## 1.2 좌측 사이드바 (FILES)

- 헤더 `FILES` + 아이콘 3개: **리스트/아웃라인 토글 · 새 문서 · 새 폴더**.
- **검색 박스**(`⌘K` 힌트) — 클릭/단축키로 팔레트 오픈.
- **프로젝트 트리**: `프로젝트명 ▾` → 폴더 접기·펼치기 → 문서. 폴더 hover 시 `⋯` 컨텍스트 메뉴. 현재 문서 하이라이트.
- **`Skills` 섹션**(별도, `+`로 추가) — OK는 에이전트 스킬을 1급 객체로 트리에 노출. *seCall엔 스킬 개념 없으면 생략 가능.*
- **구현 노트**: 폴더/문서 아이콘 구분, 현재 문서 하이라이트, 폴더 인라인 액션. seCall 볼트의 위키 폴더를 이 트리로.

## 1.3 탭 시스템

- 중앙 상단 **문서 탭 바**(`폴더/` `문서 ✕` `+`). 여러 문서 동시 열기, 탭별 닫기, `+` 새 탭.
- 폴더도 탭으로 열림(폴더 뷰 = 하나의 탭).
- **구현 노트**: 열린 탭 집합 세션 유지. 여러 위키/세션을 오가며 볼 때 유용.

## 1.4 ⌘K 커맨드 팔레트

- 통합 입력: **"Search files, folders, or commands"** + **`# By tag` 태그 필터 칩**.
- 섹션:
  - **COMMANDS**: New file(⌘N) / New folder / Open graph / Initialize starter pack.
  - **OPEN WITH AI**: `Open with AI Claude`.
- 파일·폴더·명령·태그검색·AI 진입이 한 곳(VSCode/Linear/Obsidian 표준). 마우스 없이 다 됨.
- **구현 노트(P1)**: 퍼지 검색(파일+폴더+명령) + 태그 필터 + AI 액션 라우팅. **seCall 검색이 이미 강하니 팔레트 껍데기 + 명령/AI 라우팅만 얹으면 ROI 높음.**
